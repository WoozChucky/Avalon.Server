# GC-008 Design: Span-based decrypt + merged `PacketReader.Read`

**Date:** 2026-04-17  
**Status:** Approved  
**Branch:** fix/gc-pressure

---

## Problem

Every encrypted inbound packet passes through `PacketReader.Decrypt` before `PacketReader.Read`. The current implementation produces four heap allocations:

| # | Source | Type | Fate |
|---|---|---|---|
| 1 | Protobuf-net deserializing `NetworkPacket` | `byte[] Payload` (encrypted bytes) | Replaced by #4, becomes garbage |
| 2 | `data.Take(12).ToArray()` in `AvalonCryptoSession.Decrypt` | 12-byte nonce copy | Garbage after `Decrypt` returns |
| 3 | `data.Skip(12).ToArray()` in `AvalonCryptoSession.Decrypt` | ciphertext copy | Garbage after `Decrypt` returns |
| 4 | `_encryptCipher.DoFinal(ciphertext)` | decrypted output `byte[]` | Assigned to `packet.Payload`, consumed by `Read` |

The `docs/gc-pressure.md` entry describes the "two simultaneous buffers" issue (#1 and #4); allocations #2 and #3 are additional gratuitous LINQ copies inside the crypto session. All four are eliminated by this fix.

---

## Goals

- Eliminate allocations #2, #3, and #4 entirely
- Reduce allocation #1 to the one unavoidable Protobuf deserialization (the encrypted bytes — cannot be avoided without changing `NetworkPacket.Payload`'s type, which is Approach C deferred to a future issue)
- Keep existing behaviour identical: packets that are not encrypted continue to flow through `Read` unchanged
- Keep thread-safety: `AvalonCryptoSession` already locks internally; no changes to locking strategy
- Keep existing `PacketReader` tests green; update the two `FakeAvalonCryptoSession` test doubles

---

## Non-goals

- Eliminating allocation #1 by changing `NetworkPacket.Payload` to `Memory<byte>` — deferred (Approach C), tracked as a GitHub issue
- Any changes outside the files listed in the Files Changed table
- Changing the AES-GCM cipher or key exchange protocol

## Design

### 1. `DecryptFunc` delegate — new signature

**File:** `src/Server/Avalon.Hosting/Networking/PacketReader.cs`

```csharp
// Before
public delegate byte[] DecryptFunc(byte[] data);

// After
public delegate int DecryptFunc(ReadOnlySpan<byte> input, Span<byte> output);
```

Returns the number of plaintext bytes written into `output`. The caller is responsible for supplying a sufficiently large output buffer (at minimum `input.Length` bytes — the plaintext is always shorter than the ciphertext for AES-GCM but renting `input.Length` bytes is safe and simple).

### 2. `IAvalonCryptoSession.Decrypt` — span-based signature

**File:** `src/Shared/Avalon.Common/Cryptography/IAvalonCryptoSession.cs`

```csharp
// Before
byte[] Decrypt(byte[] data);

// After
int Decrypt(ReadOnlySpan<byte> data, Span<byte> output);
```

### 3. `AvalonCryptoSession.Decrypt` — span-based implementation

**File:** `src/Shared/Avalon.Common/Cryptography/IAvalonCryptoSession.cs`

Replaces the LINQ-based implementation with span slices and BouncyCastle 2.6.2's span-capable `IBufferedCipher` API:

```csharp
public int Decrypt(ReadOnlySpan<byte> data, Span<byte> output)
{
    if (!_initialized) throw new InvalidOperationException("Crypto session not initialized");

    lock (_lock)
    {
        ReadOnlySpan<byte> nonce = data[..12];
        ReadOnlySpan<byte> ciphertext = data[12..];

        var parameters = new ParametersWithIV(new KeyParameter(_sessionKey), nonce.ToArray());
        _encryptCipher.Init(false, parameters);

        int len = _encryptCipher.ProcessBytes(ciphertext, output);
        len += _encryptCipher.DoFinal(output[len..]);
        _encryptCipher.Reset();

        return len;
    }
}
```

Note: `nonce.ToArray()` is still needed because `ParametersWithIV` accepts `byte[]` for the IV. This is one unavoidable 12-byte allocation per call; it replaces the previous 12-byte allocation so the net count is the same. The ciphertext copy (#3) and decrypted output (#4) are fully eliminated.

### 4. `IPacketReader` and `PacketReader` — merge Decrypt + Read

**File:** `src/Server/Avalon.Hosting/Networking/PacketReader.cs`

`IPacketReader.Decrypt` is removed. `Read` gains an optional `DecryptFunc?` parameter:

```csharp
// IPacketReader — before
Packet? Read(NetworkPacket packet);
void Decrypt(NetworkPacket packet, DecryptFunc decryptFunc);

// IPacketReader — after
Packet? Read(NetworkPacket packet, DecryptFunc? decrypt = null);
```

`PacketReader.Read` implementation:

```csharp
public Packet? Read(NetworkPacket packet, DecryptFunc? decrypt = null)
{
    if (!_packetTypes.TryGetValue(packet.Header.Type, out Func<ReadOnlyMemory<byte>, Packet?>? deserializer))
    {
        _logger.LogWarning("Unknown packet type {PacketType}", packet.Header.Type);
        return null;
    }

    ReadOnlyMemory<byte> payload;
    byte[]? rented = null;

    if (decrypt != null)
    {
        rented = ArrayPool<byte>.Shared.Rent(packet.Payload.Length);
        int len = decrypt(packet.Payload.AsSpan(), rented.AsSpan());
        payload = rented.AsMemory(0, len);
    }
    else
    {
        payload = new ReadOnlyMemory<byte>(packet.Payload);
    }

    try
    {
        return deserializer(payload);
    }
    finally
    {
        if (rented != null) ArrayPool<byte>.Shared.Return(rented);
    }
}
```

`packet.Payload` is never written. The rented buffer is allocated once from the pool (amortized zero allocation at steady state), used for Protobuf deserialization, and returned to the pool before the next packet begins.

### 5. `Connection.cs` — collapse two calls into one

**File:** `src/Server/Avalon.Hosting/Networking/Connection.cs`

```csharp
// Before
if (packet.Header.Flags.HasFlag(NetworkPacketFlags.Encrypted))
{
    _packetReader.Decrypt(packet, CryptoSession.Decrypt);
}
Packet? payload = _packetReader.Read(packet);

// After
Packet? payload = _packetReader.Read(
    packet,
    packet.Header.Flags.HasFlag(NetworkPacketFlags.Encrypted) ? CryptoSession.Decrypt : null);
```

### 6. `FakeAvalonCryptoSession` — updated test doubles

**Files:** `tests/Avalon.Server.Auth.UnitTests/FakeAvalonCryptoSession.cs`, `tests/Avalon.Server.World.UnitTests/FakeAvalonCryptoSession.cs`

```csharp
// Before
public byte[] Decrypt(byte[] data) => data;

// After
public int Decrypt(ReadOnlySpan<byte> data, Span<byte> output)
{
    data.CopyTo(output);
    return data.Length;
}
```

## Safety & Correctness

- **Thread safety:** `AvalonCryptoSession._lock` covers the entire cipher operation; unchanged. `ArrayPool<byte>.Shared` is thread-safe.
- **Buffer sizing:** Renting `packet.Payload.Length` bytes is always sufficient — AES-GCM plaintext is `ciphertext.Length - 16` (auth tag stripped). Over-renting by at most 28 bytes (12-byte nonce + 16-byte auth tag) per call.
- **Unencrypted packets:** The `decrypt == null` path is identical to the current `Read` path — no regression.
- **Return-to-pool safety:** The rented buffer is returned in a `finally` block; if deserialization throws, the buffer is still returned.
- **`nonce.ToArray()` residual:** One 12-byte allocation per encrypted packet remains inside `AvalonCryptoSession.Decrypt` due to `ParametersWithIV`'s `byte[]` IV parameter. This replaces the previous 12-byte LINQ alloc — net count unchanged for this item.
